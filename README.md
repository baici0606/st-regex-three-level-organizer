# 正则文件夹

`正则文件夹` 是一个 SillyTavern 前端插件，用于将全局正则、预设正则和局部正则整理到自定义文件夹中。

这版是偏稳定性的重做版：
- 保留你在酒馆里已经习惯使用的主要功能入口
- 重写内部控制器，减少多路径状态同步
- 优先降低严重 bug 和状态错乱概率

## 当前保留功能

1. 分别整理全局正则、预设正则、局部正则。
2. `未分类` 系统文件夹始终存在。
3. 新增文件夹、重命名文件夹、删除文件夹。
4. 删除文件夹及其中全部正则。
5. 通过酒馆原生拖动把正则移动到目标文件夹。
6. 通过拖动文件夹标题调整文件夹顺序。
7. 文件夹标题支持折叠和展开。
8. 文件夹级开关：可一次性启用/关闭该文件夹中的正则。
9. 文件夹导入 / 导出。
10. `未分类` 文件夹保留摇杆、开关、折叠按钮，但不可拖动排序，也不可真正导出。
11. 面板按钮区按两行组织：普通操作在第一行，删除类危险操作在第二行。

## 当前实现取舍

- 这版重点不是继续叠补丁，而是减少内部状态分叉。
- 旧版中复杂的多轮对齐、DOM fallback、深层快照回放是高风险 bug 源；本版尽量统一为“数据写入 → 刷新原生列表 → 重建分组视图”的单一主路径。
- 用户可见主功能入口仍保留，但内部语义有一处刻意简化：文件夹级开关现在采用“整组统一设为启用 / 禁用”的稳定语义，不再尝试恢复旧版那种复杂快照状态回放。
- 若你之前依赖某些非常极端的历史兼容路径（例如旧索引映射或文件夹关闭前的精细开关快照恢复），这版不会百分百复刻旧实现细节，这是为了降低严重 bug 面的工程取舍。
- 预设正则的导入与保存现在优先走真实 preset manager 接口，并在导入后做写回校验；如果当前未选中预设或接口不可用，会直接阻止导入，而不是假装成功。
- 文件夹拖拽落点现在优先按列表垂直位置推导，不再完全依赖不可见 anchor 的命中结果，因此把正则拖进其他文件夹、未分类和折叠文件夹会更稳定。
- 拖拽结果会在原生 regex 列表执行重载前先按目标文件夹落盘；插件不再在原生 `loadRegexScripts()` 之后用未分组 DOM 反向覆盖 assignment，避免拖拽后回到未分类。
- 正则条目现在可以从左侧拖拽图标和名字区域发起拖拽，同时文件夹落点区域已放大，降低拖进各种文件夹时的精度要求。

## 使用方式

1. 打开正则面板。
2. 点击 `新增文件夹`。
3. 输入文件夹名称。
4. 通过酒馆原生拖动，把正则拖到目标文件夹下。
5. 点击文件夹标题可折叠或展开对应内容。
6. 使用文件夹级开关统一启用/关闭文件夹中的正则。
7. 使用顶部按钮导入文件夹，或使用普通文件夹标题栏中的导出按钮导出文件夹。

## 说明

- 当前实现是普通文件夹，不是树形多级目录。
- 文件夹信息保存在浏览器本地 `localStorage` 中，并按 global / preset / scoped 分 scope 隔离。
- 不修改正则脚本结构本身，只管理分组与排序落点。
- `未分类` 是系统保底文件夹：始终显示，名称固定，不参与文件夹排序，但仍可作为拖拽落点。

## 安装

在 SillyTavern 的“安装拓展”里输入：

```text
https://github.com/baici0606/st-regex-three-level-organizer
```

## 手动安装

把整个 `st-regex-three-level-organizer` 文件夹放到：

```text
SillyTavern/public/scripts/extensions/third-party/
```

然后在 SillyTavern 扩展管理中启用。
